httpproxy2socket

Date Fri 03 July 2015 By Seanchann Zhou Category network Tags httpproxy socket

系统：Fedora 22

这个需求的来源是无法通过shadowsocket的方式给其他的应用提供http proxy。大量的程序目前都是可以配置http proxy，而不是socket4或者socket5的方式。因此促使我做了一下这方面的工作，把我的经验分享给大家。

前提条件：

  • shadowsocket已经配置完成并可用

方案描述

我的方案就是搭建socket5代理、http代理，最终通过proxychains将这个两个代理连接。

socket5的代理就是我们需要提前配置好的shadowsocket的环境，它给本地提供了一个默认端口为1080的socket5的代理。这个代理是可以让我们自由的畅享Internet的。。。

http proxy我们通过squid来实现

proxychains主要用组装我们的代理链条。

squid

squid可以配置为一个http proxy server，并且有相关的权限控制。我们用它来作为本地的http proxy。

安装squid：

dnf install squid

配置squid：

vi /etc/squid/squid.conf

我们增加一些必要的基本配置：

acl http_proxy src 192.168.1.0/24
http_access allow http_proxy

上面的这个配置许可192.168.1.0/24网络内对此http proxy的访问。 在配置文件中，也可以看到默认的监听端口是3182

使能开机启动以及启动服务：

systemctl enable squid.service
systemctl start squid.service

注意：如果你使能了iptables，那么配置3128端口的路由规则

proxychains

proxychains可以为我们提供强大的代理链条。我们来配置我们的http proxy再连接到socket5。完成我们最初的设想。

安装proxychains：

dnf install  proxychains

proxychains的配置文件的查找优先级：

1 ./proxychains.conf 2 $(HOME)/.proxychains/proxychains.conf 3 /etc/proxychains.conf **

推荐在用户目录创建

完成以后，我们配置proxychains.conf，增加如下的内容：

[ProxyList]
# add proxy here ...
# meanwile
# defaults set to "tor"
http 192.168.1.2 3128
socks5  192.168.1.2 1080

上面的配置文件的含义是，请求首先进入http proxy，随后再通过socket5到达目的地。

使用范例：

proxychains wget www.test.com
