mongodb---user

Date Sun 07 June 2015 By Seanchann Zhou Category mongodb Tags mongodb

系统：CentOS7 + docker 1.6

mongodb版本： 3.0.2

安装工作完成后，整个的mongodb运行起来后，在开始工作前，首先应该为我们的mongodb配置适当的用户以及其权限。

注意下，网上大多数的直接进入命令行,执行：

use admin
db.adduser("test","123")

上面这种方式在3.0的版本上是行不通的

添加管理员用户

通过docker首先启动不带--auth的mongodb：

docker run --volumes-from=mongodb_data --name mongodb -d mongo

更详细的对安装的描述

进入mongo shell

添加管理员用户：

use admin
db.createUser(
{
      user: "admin",
      pwd: "admin",
      roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

roles字段的说明：添加管理员用户的时候，提供了两种roles：userAdmin以及userAdminAnyDatabase

  • userAdmin：提供去创建和修改当前数据库的roles以及user。这个role也间接的提供了superuser给我们去访问数据库或者其他，如果当前数据库是admin 数据库，cluster数据库。userAdmin role允许用户去更改其他用户的权限，当然也包括他们自己。它支持操作有：

    • changeCustomData
    • changePassword
    • createRole
    • createUser
    • dropRole
    • dropUser
    • grantRole
    • revokeRole
    • viewRole
    • viewUser

  • userAdminAnyDatabase: 这个role与userAdmin支持的权限类似，但是它应用的对象是在整个集群中的数据库。这个role也提供了下面可以在集群中使用的操作：

    • authSchemaUpgrade
    • invalidateUserCache
    • listDatabases

检查创建的用户

根据前面的命令，我们的应该有了一个admin的用户，执行命令检查下：

show users

我们得到的输出是：

{
        "_id" : "admin.admin",
        "user" : "admin",
        "db" : "admin",
        "roles" : [
                {
                        "role" : "userAdminAnyDatabase",
                        "db" : "admin"
                }
        ]
}

同样的，我们也可以使用命令：

  db.system.users.find()

重新启动mongodb with auth parameter

接下来，我们重新启动一个mongodb，使用--auth：

docker run --volumes-from=mongodb_data --name mongodb -d mongo --auth

执行下面的命令，检查鉴权是否可以成功：

use admin
db.auth("admin","admin")

执行后返回的结果是：

1

这表示我们的执行用户鉴权是成功的

到此，管理数据库用户的用户admin创建成功，接下来就是创建针对每个数据库的用户

创建数据库用户

我们先来执行下面的命令，创建一个数据库test：

use test

创建test数据库的用户test,这里给的role是readWrite：

db.createUser({user:"test", pwd:"test", roles:[{role:"readWrite",db:"test"}]})

我们对用户进行下鉴权：

db.auth("test","test")

如果返回为：

1

那对test数据库创建的test用户是ok的，随后在test的数据库中，就可以使用test用户对此数据库进行读写。
