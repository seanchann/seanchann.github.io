在这篇文章中，我们会创建一个搜索iTunes上的音乐的应用

我们会使用到免费的API:https://affiliate.itunes.apple.com/resources/documentation/itunes-store-web-service-search-api/

学习目标

  • CORS是什么，如何在浏览器中临时的旁路这个安全特性。
  • 如何创建一个中间服务，来处理我们的到API端点的HTTP请求
  • 如何提供一个服务并且在组件中使用
  • 如何将应答转化为一个domain model的实例
  • 如何通过使用promise来处理异步的任务

跨域资源共享

为了使用这个API我们需要去处理CORS的问题，也就是跨域共享。

这是一个所有的浏览器都支持的安全措施，来避免你可能直接的访问API，当然包括iTunes api，都是通过这个方式来保护。

由于CORS的原因如果我们尝试去发起请求到iTunes API。浏览器将会报CORS的错误。

Chrome

针对Chrome，我偏好使用插件来控制CORS的开关。安装这个插件https://chrome.google.com/webstore/detail/allow-control-allow-origi/nlfbmbojpeacfghkpbjhddihlkkiljbi

这个插件将会添加一个按钮到chrome上，我们可以点击按钮来切换是否开启或者关闭CORS

Firefox

如果使用FireFox安装这个插件https://addons.mozilla.org/en-Gb/firefox/addon/cors-everywhere/

我们有一个input和search button。当某人按下button，我们会执行doSearch函数，来搜索iTunes数据库。

创建一个搜索服务

到目前为止，我们已经知道了如何注入http到组件的构造函数中，然后直接使用http client库来发起请求。

然后推荐的方法是创建一个中间服务层来使用HTTP client库来发起请求。取代原先返回的原始json数据为转换过的类实例，我们也称之为domain model

这大大的方便了我们去测试组件。我们没有模拟原始的HTTP请求，我们定义了domain model实例的测试数据来进行测试。

因此，我们创建了我们的中间层服务，我们称之为SearchService

@Injectable() 
class SearchService {
  apiRoot:string = 'https://itunes.apple.com/search';
  results:Object[];
  loading:boolean;

  constructor(private http:Http) { 
    this.results = [];
    this.loading = false;
  }

  search(term:string) {
  }
}

我们使用@Injectable来装饰了我们的类，因此Angular会知道，需要注入Http的服务到我们的构造函数中。可以看到构造函数中对http的注入。

使用promises

search函数用来完成一个异步的调用，使用http client库来发起到iTunes API的请求。

在前面的章节我们知道，有两种方式来处理异步函数，Promises和Observables

在本章节中，我们使用Promises，在下一个章节中，我们使用Observables

由于我们使用promise，因此需要search函数来返回一个promise，例如：

search(term:string) {
  let promise = new Promise((resolve, reject) => {
    //TODO
  });
  return promise;
}

当从iTunes API获取到了http的应答，我们完成了我们的处理调用resolve()函数。这使得我们可以明确异步任务完成，并执行任何后续任务。

如果http的应答返回了的错误,我们调用reject(),这个明确的指出这里发生了错误.

因此让我们来完善下面的get请求,例如:

let promise = new Promise((resolve, reject)=>{
  let apiURL = `${this.apiRoot}?term=${term}&media=music&limit=20`;
  this.http.get(apiURL)
    .toPromise()
    .then(
      res =>{
        console.log(res.json());
        resolve();
      }
    );
});

首先我们构建了请求的URL,例如:

let apiURL = `${this.apiRoot}?term=${term}&media=music&limit=20`;

然后发起了get请求:

this.http.get(apiURL)

它会返回一个Observable,所以我们将其转换为Promise

.toPromise()

然后我们打印出应答对象:

.then(
  res => { // Success
  console.log(res.json());
  }
);

使用search服务

需要配置SearchService作为一个provider在我们的NgModule

@NgModule({
  imports: [
    BrowserModule,
    HttpModule,
  ],
  declarations: [AppComponent],
  bootstrap: [AppComponent],
  providers: [SearchService]
})
class AppModule { }

也需要去注入我们的search服务到AppComponent,然后当用户按下按钮的时候,调用itunes.search()

class AppComponent {
  constructor(private itunes:SearchService) { }

  doSearch(term:string) {
    this.itunes.search(term)
  }
}

我们只对请求的结果感兴趣,它是一个搜索返回结果列表.

不用将搜索结果返回到组件，而是将搜索结果存储在SearchService里，以便更轻松地在组件之间共享结果，同时添加错误处理程序，如下所示：

search(term:string) {
  let promise = new Promise((resolve, reject) => {
    let apiURL = `${this.apiRoot}?term=${term}&media=music&limit=20`;
    this.http.get(apiURL)
      .toPromise()
      .then(
        res => { // Success
        this.results = res.json().results;
        resolve();
        },
        msg => { // Error
        reject(msg);
        }
      );
  });
  return promise;
}
}

我们调用了resolve()以及reject()函数来完成这个异步的调用当获得从服务器返回应答的时候.

现在让我们渲染这些结果到我们的组件上.

数据保存在iTunes服务中,我们使用NgFor命令来遍历结果

<ul>
  <li *ngFor="let track of itunes.results">
    {{track.trackName}}
  </li>
</ul>

只显示名字显得单调了些,让我们显示一些图片上去

<ul class="list-group">
  <li class="list-group-item"
      *ngFor="let track of itunes.results">
    <img src="{{track.artworkUrl30}}">
    <a target="_blank"
       href="{{track.trackViewUrl}}">{{ track.trackName }}
    </a>
  </li>
</ul>

添加加载指示

现在基本上工作都已经完成,只有在异步请求的时候,我们需要一个加载指示器给终端用户,以表示当前程序运行中而不是程序异常了.

让我们显示Loading...字符,但我们等待请求应答结果的时候.

我们创建一个loading的布尔变量载我们的组件内,然后有条件的显示ptag如果loading是true,例如:

<div class="text-center">
  <p class="lead" *ngIf="loading">Loading...</p>
</div>

我们将loading初始化为false,当调用doSearch()的时候,设置为true.

class AppComponent {
  private loading: boolean = false;

  constructor(private itunes:SearchService) { }

  doSearch(term:string) {
    this.loading = true;
    this.itunes.search(term)
  }
}

当处理完成后,我们重新将loading置为false

doSearch(term:string) {
  this.loading = true;
  this.itunes.search(term).then( () => this.loading = false)
}

使用 domail model

我们现在已经有了一个完整功能的APP

然而我们存储的数据现在还是一个文本的json对象,一个好的做法是解析这些原始的数据到一个类实例,也叫做 domain model

首先,我们创建一个类叫做SearchItem

class SearchItem {
  constructor(public name: string,
              public artist: string,
              public link: string,
              public thumbnail: string,
              public artistId: string) {
  }
}

然后在SearchService中我们将数据解析到这个类的实例中.

我们使用map来完成这个操作

this.results = res.json().results.map(item => { 
  return new SearchItem(
      item.trackName,
      item.artistName,
      item.trackViewUrl,
      item.artworkUrl30,
      item.artistId
  );
});

完整的SearchService代码清单如下:

@Injectable()
export class SearchService {
  apiRoot: string = 'https://itunes.apple.com/search';
  results: SearchItem[];
  loading: boolean;

  constructor(private http: Http) {
    this.results = [];
    this.loading = false;
  }

  search(term: string) {
    let promise = new Promise((resolve, reject) => {
      let apiURL = `${this.apiRoot}?term=${term}&media=music&limit=20`;
      this.http.get(apiURL)
          .toPromise()
          .then(
              res => { // Success
                this.results = res.json().results.map(item => {
                  return new SearchItem(
                      item.trackName,
                      item.artistName,
                      item.trackViewUrl,
                      item.artworkUrl30,
                      item.artistId
                  );
                });
                // this.results = res.json().results;
                resolve();
              },
              msg => { // Error
                reject(msg);
              }
          );
    });
    return promise;
  }
}

results属性现在就是SearchItem的数组

更新下我们的显示逻辑

<ul class="list-group">
  <li class="list-group-item"
      *ngFor="let track of itunes.results">
    <img src="{{track.thumbnail}}">
    <a target="_blank"

总结

推荐的方式是构建一个中间服务层来完成所有的http请求,然后将返回结果从原始数据转换为domain models

在本章中,我们使用promise来除了异步的请求.并且通过我们自己的SearchService返回了一个promise
