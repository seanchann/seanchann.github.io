cmake 路径检索

2017-08-18
build-system

文章目录

    • Modules路径检索

在实际的开发过程中，经常需要配置库的检索路径，头文件的检索路径，工程的*Config.cmake检索路径，自定义的FindProject.cmake路径等等。因此，cmake的路径检索以及优先级策略等等，都是编写cmake工程文件所必需的知识，本文整理了cmake的路径检索知识，希望对大家有所帮助。

Modules路径检索

cmake的模块目录是在安装cmake的时候直接安装好的（针对linux，默认在/usr/share/cmake/Modules/），这个目录通常情况下，我们不要去更改这个目录下面的任何内容。

如果想要自定义一些*.cmake的文件来使用。那么通常的做法是针对这个工程来添加自定义的cmake文件。然后安装到${prefix}/*目录。如下所示，find_package或者其他的函数可以在下面目录中来检索自定义的*.cmake文件

<prefix>/                                               (Windows)
<prefix>/(cmake|CMake)/                                 (Windows)
<prefix>/<name>*/                                       (Windows)
<prefix>/<name>*/(cmake|CMake)/                         (Windows)
<prefix>/(lib/<arch>|lib|share)/cmake/<name>*/          (Unix)
<prefix>/(lib/<arch>|lib|share)/<name>*/                (Unix)
<prefix>/(lib/<arch>|lib|share)/<name>*/(cmake|CMake)/  (Unix)
<prefix>/<name>.framework/Resources/                    (Apple)
<prefix>/<name>.framework/Resources/CMake/              (Apple)
<prefix>/<name>.framework/Versions/*/Resources/         (Apple)
<prefix>/<name>.framework/Versions/*/Resources/CMake/   (Apple)
<prefix>/<name>.app/Contents/Resources/                 (Apple)
<prefix>/<name>.app/Contents/Resources/CMake/           (Apple)
